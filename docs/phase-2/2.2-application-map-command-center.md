## Purpose
- Map the authenticated command center experience to operational domains and key workflows.
- Define role-aware navigation and system touchpoints.

## Scope
- Covers `/admin/*` command center surfaces and supporting APIs.
- Excludes public marketing pages and deep UI visual design details.

## Key decisions
- Command center is organized by operational responsibility: intake, detection, incidents, escalation, SLA, IAM, audit, newsroom.
- API surface aligns one-to-one with bounded contexts where feasible.
- Admin APIs require authenticated session checks and tenant-aware request context.
- Audit visibility is exposed as a first-class command center function.

## Application map (authenticated)
- `/admin/login`
- Intent: privileged entry point.
- Primary domain: IAM.
- Core actions: token/session establishment, redirect to operational hub.

- `/admin/inbox`
- Intent: triage inbound contact messages and assign ownership.
- Primary domains: Contact Intake, IAM, Audit.
- Core actions: filter/search, status updates, assignment, CSV export.
- Supporting APIs: `/api/admin/inbox`.

- `/admin/alerts`
- Intent: monitor active detection stream.
- Primary domains: Threat Monitoring, Audit.
- Core actions: review severity, update status, correlate to incident.
- Supporting APIs: `/api/admin/alerts`.

- `/admin/incidents`
- Intent: manage incident lifecycle and severity handling.
- Primary domains: Incident Response, SLA, Audit.
- Core actions: status transition, ownership review, SLA profile awareness.
- Supporting APIs: `/api/admin/incidents`.

- `/admin/on-call`
- Intent: operational coverage and escalation readiness.
- Primary domains: On-Call & Escalation, IAM.
- Core actions: view active schedule windows and policy bindings.
- Supporting APIs: `/api/admin/on-call`.

- `/admin/sla`
- Intent: contractual response posture and breach visibility.
- Primary domains: SLA & Breach Detection, Compliance, Audit.
- Core actions: review profiles, inspect breaches.
- Supporting APIs: `/api/admin/sla`.

- `/admin/iam`
- Intent: tenant-scoped identity and access governance summary.
- Primary domains: IAM, Compliance.
- Core actions: view teams, orgs, role grants.
- Supporting APIs: `/api/admin/iam`.

- `/admin/audit`
- Intent: immutable event review for forensics/compliance.
- Primary domains: Audit & Event Backbone, Compliance.
- Core actions: event review, filtered lookup via API.
- Supporting APIs: `/api/admin/audit/events`.

- `/admin/newsroom`
- Intent: intelligence publishing operations.
- Primary domains: Newsletter/Intelligence Publishing, Audit.
- Core actions: subscriber and campaign overview, campaign scheduling.
- Supporting APIs: `/api/admin/newsroom/campaigns`.

- `/admin/diagnostics`
- Intent: operational health indicators for stores/config.
- Primary domains: Platform Operations.
- Core actions: runtime/store readiness checks.

## Role mapping (initial)
- `Platform Admin`
- Full access across IAM, diagnostics, and governance exports.
- `SOC Analyst`
- Alerts, incidents, inbox triage, restricted IAM read.
- `Incident Commander`
- Incident lifecycle and escalation controls, SLA operational read/write.
- `Compliance Lead`
- Audit review/export, SLA breach oversight, governance reporting.
- `Newsroom Operator`
- Subscriber/campaign management and publication cadence controls.

## Workflow map (high-level)
- Intake to response:
- Contact message created -> triaged -> assigned -> optionally linked to incident.
- Detection to closure:
- Alert created -> investigated -> escalated -> incident opened -> status transitions -> postmortem.
- SLA and escalation:
- Incident opens clock -> warning threshold -> breach record -> compliance review.
- Publishing:
- Subscriber captured -> campaign scheduled -> dispatch event logged.

## Assumptions (explicit)
- Current command center is GEM-operated and not customer-admin self-hosted.
- Tenant context is carried per request and must be respected by all admin APIs.
- Role catalog above is initial and expected to expand.

## Non-blocking open questions

- Whether incident timeline editing is permitted after closure.
- Whether diagnostics should expose dependency health beyond local stores in v0.2.
